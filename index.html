Project Structure

The way in which this project behaves is a little strange, so some explanation is necessary.

The core business logic of this project is written in Java. The application works by letting the JVM continuously run the exported .jar file. The .jar then, indefinitely listens to it's STDIN in a blocking fashion [on purpose; That way the JVM isn't going crazy in an endless loop]. Upon receiving input, which must be pre-formatted JSON, the running Java application will handle the request, and eventually return a response by writing a JSON message to STDOUT.

A Node.js application is also running. Node.js acts as a bridge pattern between the Java process and the client. Node.js actually starts and manages the Java JAR. Upon receiving an HTTP request, node forwards the request to the JVM.

The Node.js process is essentially acting as the bridge between the client and the business logic.

Patterns Used

I used many GOF design patterns in my project. Some of the more prevalent ones were the singleton pattern, interpreter pattern, and strategy pattern.

The Singleton pattern is used to manage the database instance, so it doesn't create multiple instances to the same database object.

The Interpreter pattern is used to interpret requests, and then respond to them.

The Strategy pattern goes hand-in-hand with the Interpreter pattern in this case. I have two strategies for interpreting commands. They are a JSON-formatted strategy, and a CLI-formatted strategy.

Commands

The Java program handles all the business logic associated with Coplan, by communicating with a Node.js process via JSON-formatted commands sent through STDIN/STDOUT.

Commands are to be formatted in the following way:

  • There are two essentail components: 'route', and 'data'
  • 'route' specifies the endpoint you want to talk you
  • 'data' specifies an object with keys and values pertinant to the endpoint
  • If you attempt to access an endpoint without the needed requirements, an error message will be returned

Request Example

{ 
   route: "createUser", 
   data: {
      username: "bshutt",
      firstName: "Brady",
      lastName: "Shutt",
      major: "Computer Science"
   }
}
